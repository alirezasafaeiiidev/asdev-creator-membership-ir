# راهنمای نگارش و استاندارد مستندات (Documentation Style Guide)
نسخه: 1.0
تاریخ: 2026-02-08

هدف: یکدست‌سازی نگارش، ساختار و کیفیت مستندات.

---

## 1) قواعد نگارش
- جملات کوتاه، مستقیم و غیرمبهم
- واژه‌های دستوری:
  - «باید» (Must) / «نباید» (Must Not) / «پیشنهاد می‌شود» (Should)
- نام فایل‌ها/مسیرها همیشه انگلیسی و داخل backtick

## 2) Header استاندارد هر سند
هر سند باید با عنوان + نسخه شروع شود:
```
# عنوان
نسخه: 1.0
```

## 3) استاندارد فنی داخل سند
- snippetها قابل کپی و اجرا
- اسکریپت‌های جدی: `set -euo pipefail`
- ممنوع: اشاره به CDN یا سرویس خارجی runtime

## 4) ADR
هر تصمیم معماری مهم باید ADR داشته باشد:
- `docs/ARCHITECTURE_DECISIONS/ADR-TEMPLATE.md`

## 5) معیار قبولی مستندات
- قابل اجرا + قابل ممیزی + هماهنگ با سایر اسناد
- پاس شدن `pnpm -w docs:validate`

پایان.
